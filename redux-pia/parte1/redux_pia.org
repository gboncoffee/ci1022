* Sinais de controle
- Tipo: Se I, a entrada ~ra~ do banco de registradores deve receber 0; ~rb~
  deve receber 3. Se R, as entradas recebem o valor dos seus campos na
  instrução. Se I, a entrada ~b~ da ula deve receber o imediato com sinal
  extendido.
- rw: Se a, a entrada ~rw~ do banco de registradores deve receber o valor do
  campo ra da instrução. Se 0, deve receber 0. Se 3, deve receber 3.
- ld: Se ativa, a entrada ~d~ do banco de registradores deve receber a saída
  da memória.
- rwe: Conectada diretamente à entrada ~we~ do banco de registradores.
- mwe: Conectada diretamente à entrada ~mwe~ da memória.
- +pc: Se +sI, o PC deve ser somado ao imediato com sinal. Se -uI, deve
  ser decrementado do imediato somado à 1 sem sinal. Se 1, deve ser somado
  à 1.
- loop: Se ativa, a entrada ~a~ da ula deve ser -1. Caso o resultado da ula
  seja diferente de zero, PC deve receber a subtração de si próprio pelo
  imediato sem sinal + 1.
- b: Se ativa e a saída da ula for diferente de 0, o PC deve ao invés da
  soma, receber o valor da saída ~b~ do banco de registradores.
- acc: Conectada diretamente à entrada ~acc~ do banco de registradores.
- func: Função a ser computada na ula.

** Codificações
- Tipo: 1 bit, R = 0, I = 1.
- rw: 2 bits, a = 0, 0 = 1, 3 = 2.
- +pc: 1 bit, 1 = 0, +sI = 1.
- func: 4 bits:
  - add = 0.
  - sub = 1.
  - not = 2.
  - and = 3.
  - or = 4.
  - xor = 5.
  - << = 6.
  - >> = 7.
  - mul = 8.
  - ldui = 9.

* Instruções extendidas
** loop
A instrução decrementa o registrador ~r3~ e, caso o resultado seja diferente
de zero, desvia o programa subtraindo o imediato sem sinal + 1 do PC.

** mac
A instrução multiplica os valores de ~ra~ e ~rb~, guardando o resultado em ~r0~.

** ldui
A instrução concatena a parte baixa de ~r0~ com o imediato na parte alta.

* Notas do projeto
** ~acc~
O banco de registradores possui uma entrada adicional ~acc~ que, caso ativa,
soma o valor a ser guardado no registrador ao invés de substituí-lo.

* Assembly

O programa que faz soma vetorial:

#+BEGIN_SRC asm
	xor r0, r0
	xor r1, r1
	xor r2, r2
	xor r3, r3

	addi 5
	addi 5

	add r1, r0

	xor r0, r0
	addi 7
	ldui 1

	add r2, r0

	xor r0, r0
	addi 3

	add r3, r0
	xor r0, r0
	addi 1
for:
	st r1, r2

	add r1, r0
	add r2, r0

	loop for

	xor r0, r0
	addi 1
;; ji 0
	ecall
#+END_SRC

O programa que faz produto escalar de dois vetores:

#+BEGIN_SRC asm
	xor r0, r0
	xor r1, r1
	xor r2, r2
	xor r3, r3

	addi 5
	add r3, r0

	xor r0, r0
for:
;; Por conveniência, vamos usar a posição 0 da memória para guardar o
;; acumulador. Esse loop tem exatamente o tamanho limite para usar a instrução
;; loop.
	xor r1, r1
	st r0, r1

	xor r0, r0
;; Os vetores ficam em 0x20 para salvar uma instrução aqui e manter o loop no
;; tamanho para utilizar a instrução loop.
	ldui 2
	add r0, r3
	xor r2, r2
	add r1, r0
	add r2, r0
	xor r0, r0
	addi 5
	add r2, r0
	ld r1, r1
	ld r2, r2

	xor r0, r0
	ld r0, r0

	mac r1, r2

	loop for

	xor r0, r0
	addi 1
;; ji 0
	ecall

.space 5
a:
.bits8 1
.bits8 2
.bits8 3
.bits8 4
.bits8 5
b:
.bits8 5
.bits8 4
.bits8 3
.bits8 2
.bits8 1
#+END_SRC
